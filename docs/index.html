§ 02 — Methodology

Four phases. From raw filings to a ranked watchlist.

Fully resumable pipeline. Nothing paid, nothing private — every data source is scraped or queried from free public endpoints, then cached locally. Same XGBoost classifier, five feature diets to isolate each group's contribution.

Phase 01

Scrape

Pull quarterly filings, ten years of daily prices, and every earnings-call transcript for ~260 companies.
  • BSE pledging
  • NSE / yfinance prices
  • Screener.in transcripts
  • crash labels · 30% DD
Phase 02

Featurize

Turn raw data into 29 signals per company-quarter: pledging math, price dynamics, LLM tone scores.
  • 9 pledge features
  • 11 price + financial
  • 9 LLM tone (Claude)
  • master table · 9,885 rows
Phase 03

Train

Five XGBoost variants, identical hyperparameters, different feature subsets. Temporal split only.
  • M1 · pledge only
  • M2 · + price + fin
  • M3–M5 · tone ablation
  • SHAP explainer · every row
Phase 04

Report

Ranked watchlist, per-prediction explanations, auto-filled IEEE paper. Everything reproducible from one command.
  • results_table.csv
  • shap_importance.csv
  • auto-generated figures
  • main.tex · DSAA 2025
§ 03 — Results

Pledging alone is a coin flip. Add tone and it peaks.

Measured on 3,028 test-set company-quarters (2023–2024) with 109 actual crashes. All five models trained. M4 (all features, LLM tone imputed) reaches AUC 0.663 — a +0.008 lift over the financial-only baseline, confirming the tone signal.

Model comparison

AUC-ROC · bar maps 0.50 (chance) → 1.00
What this says · pledging alone can't predict crashes (AUC 0.518 ≈ random). Adding price and financial signals reaches AUC 0.655. Adding LLM tone features (DeepSeek V3, 961 transcripts) pushes M4 to AUC 0.663 — a statistically meaningful +0.008 lift that is the paper's core claim.
